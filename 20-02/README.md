## Домашнее задание к занятию «Микросервисы: принципы» FOPS-38 (Щербатых А.Е.)

Вы работаете в крупной компании, которая строит систему на основе микросервисной архитектуры. Вам как DevOps-специалисту необходимо выдвинуть предложение по организации инфраструктуры для разработки и эксплуатации.

### Задача 1: API Gateway

Предложите решение для обеспечения реализации API Gateway. Составьте сравнительную таблицу возможностей различных программных решений. На основе таблицы сделайте выбор решения.

Решение должно соответствовать следующим требованиям:

- маршрутизация запросов к нужному сервису на основе конфигурации;
- возможность проверки аутентификационной информации в запросах;
- обеспечение терминации HTTPS;

Обоснуйте свой выбор.

### Ответ 1.

Для выбора оптимального инструмента были рассмотрены наиболее популярные решения, соответствующие запрошенным техническим требованиям.

| Характеристика	| NGINX	| Kong Gateway	| Apache APISIX |
|-----------------|-------|---------------|---------------|
| Маршрутизация	| Статическая/динамическая (модули) |	Высокодинамическая |	Высокодинамическая |
| Аутентификация |	Плагины/Lua-скрипты	| Встроенные плагины (JWT, OAuth2) |	Встроенные плагины (JWT, OAuth2) |
| Терминация HTTPS |	Да |	Да	| Да |
| Хранение конфигурации	| Файлы |	PostgreSQL / Cassandra	| etcd (распределенное, надежное) |
| Расширяемость |	Средняя	| Высокая (плагины)	| Очень высокая (плагины)|
| Производительность (QPS) | Очень высокая |	Высокая	| Очень высокая |

Обоснование выбора

Для современной микросервисной архитектуры рекомендуется внедрение Kong Gateway или Apache APISIX. Оба решения являются «cloud-native» продуктами, спроектированными специально для высоконагруженных распределенных систем.

Выбор обусловлен следующими факторами:

**Динамическая конфигурация**: В отличие от классического NGINX, требующего перезагрузки конфигурации при изменении маршрутов, Kong и APISIX поддерживают обновление правил «на лету» через API, что критически важно при частом деплое микросервисов.

**Экосистема плагинов**: Оба решения предоставляют готовые плагины для проверки JWT-токенов, интеграции с внешними Identity Provider'ами (OIDC, OAuth2) и управления лимитами запросов (rate limiting), что полностью покрывает требования к безопасности.

**Терминация HTTPS**: Все указанные решения надежно справляются с терминацией SSL/TLS, однако специализированные шлюзы (Kong/APISIX) позволяют удобнее управлять сертификатами в автоматизированных пайплайнах.

Учитывая зрелость сообщества и широкий спектр коммерческой и Open Source поддержки, в качестве базового решения для компании стоит выбрать Kong Gateway как наиболее сбалансированный вариант между мощностью, простотой настройки и поддержкой стандартов безопасности.

### Задача 2: Брокер сообщений

Составьте таблицу возможностей различных брокеров сообщений. На основе таблицы сделайте обоснованный выбор решения.

Решение должно соответствовать следующим требованиям:

- поддержка кластеризации для обеспечения надёжности;
- хранение сообщений на диске в процессе доставки;
- высокая скорость работы;
- поддержка различных форматов сообщений;
- разделение прав доступа к различным потокам сообщений;
- простота эксплуатации.

Обоснуйте свой выбор.

### Ответ 2.
